init

git init

Creates new .git repository folder in the current folder.

status

git status

Returns status of current branch including:

  • Untracked files: files which are have not being added to the repository.
  • Unstaged files: files that are tracked but which are have not been staged.
  • Staged files: files that are staged and will be on next commit.

add

git add <file1 name> [file2 name] ...

  1. Adds specified file(s) to repository branch if file is not currently being tracked.

  2. Stages modified files that are alreay part of current repository branch.

Separate files with spaces.

Sepecifying a directory will add all files to next commit.

git add -A

Adds all changes including new files, modified files and removed files in current directory.

git add .

Adds all changes including new files, modified files and but not removed files in current directory.

diff

git diff

Compares what has been changed but is not staged yet.

git diff --staged

Compares staged changes to last commit.

commit

git commit

Launches text editor so commit message can be typed in. Description can be added by leaving blank line after commit message.

git commit -m

Allows inline entry of commit message. (doesn't launch text editor.)

commit -a -m

Skips staging and commits all current changes with an inline commit message.

rm

git rm <file_name> [directory]

Removes file [directory] from working directory and repository.

git rem --cached <file_name>

Removes file from repository but not from working directory.

git commit --amend

Takes previous commit and returns it to staing area.

log

git log

Outputs commit history

git log -(n)

Restricts out put to nth last commits.

git log --since="yyyy-mm-dd" (--after="yyyy-mm-dd")

Limits output to commits made --after or --since aspecified date.

git log --until="yyyy-mm-dd" (--before="yyyy-mm-dd")

Limits out put to commits made before specified date.

git log -S<string>

Restricts output to commits with specified string.

reset

git reset <file_name>

Removes specified files from staging.

remote

git remote add origin (url)

Adds remote repository, located at specified URL, to current local repository.

git push -u origin master

Pushes current local branch to master branch on remote repository.

git remote -v

Displays current remote push and fetch URLs.
